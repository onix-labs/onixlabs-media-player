Welcome to ONIXPlayer

ONIXPlayer is a desktop media player built with Electron that supports a wide range of audio and video formats with real-time audio visualizations.

Quick Start

  1. Install dependencies — Open Settings (Cmd/Ctrl + ,) and navigate to Dependencies to install FFmpeg and optionally FluidSynth for MIDI support.
  2. Open media files — Use File > Open (Cmd/Ctrl + O) or drag and drop files onto the player window.
  3. Control playback — Use the transport controls at the bottom of the window or keyboard shortcuts (Space to play/pause).
  4. Explore visualizations — Audio files display real-time visualizations. Switch between modes via View > Visualizations in the menu.

First-Time Setup

ONIXPlayer requires FFmpeg for audio and video playback. Without it, no media files can be opened. Navigate to Settings > Dependencies and click "Install" next to FFmpeg.

For MIDI file playback, you also need FluidSynth and at least one SoundFont (.sf2) file. Install FluidSynth from Settings > Dependencies, then install a SoundFont file.

Desktop Mode

The default window mode. Shows the full player interface with header, visualization or video content area, playlist sidebar, and transport controls at the bottom. The window can be freely resized (minimum 640x480).

Fullscreen Mode

Hides the window chrome and expands the content to fill the entire screen. Transport controls auto-hide after a configurable delay and reappear when the mouse is moved.

  • Enter fullscreen: Click the expand button in the controls, or use Ctrl+Cmd+F (macOS) / F11 (Windows/Linux)
  • Exit fullscreen: Press Escape, or move the mouse and click the compress button
  • Configure auto-hide delay: Settings > Application > Controls Auto-Hide

Miniplayer Mode

A compact, always-on-top window (320x200 to 640x400) that stays above other windows. Ideal for keeping playback visible while working in other applications.

  • Enter miniplayer: Click the miniplayer button in the controls
  • Drag to move: Click and drag anywhere on the miniplayer window
  • Edge snapping: The window magnetically snaps to screen edges when dragged nearby
  • Resize: Drag the window edges (constrained to 320x200 minimum, 640x400 maximum)
  • The miniplayer position and size are remembered between sessions

About Dependencies

ONIXPlayer relies on external tools for media processing. These are managed through the Settings > Dependencies panel, which can automatically install and uninstall them via your system package manager (Homebrew on macOS).

FFmpeg

FFmpeg is the primary media processing engine. It handles transcoding, format detection, and metadata extraction for all audio and video formats. Without FFmpeg, no media files can be played.

  • Required for all audio formats (MP3, FLAC, WAV, OGG, M4A, AAC, WMA)
  • Required for all video formats (MP4, M4V, MKV, AVI, WebM, MOV)
  • Handles real-time transcoding for non-native browser formats
  • Install via Settings > Dependencies

FluidSynth

FluidSynth is a software synthesizer that renders MIDI files into audible audio using SoundFont instrument samples. It is only needed for MIDI file playback.

  • Required only for MIDI (.mid, .midi) files
  • Requires at least one SoundFont (.sf2) file to produce sound
  • Install via Settings > Dependencies

SoundFonts

SoundFont (.sf2) files contain instrument samples used by FluidSynth to synthesize MIDI audio. You need at least one SoundFont installed for MIDI playback to produce sound.

  • Install SoundFonts via Settings > Dependencies (under the FluidSynth section)
  • Popular free SoundFonts include FluidR3, GeneralUser GS, and Arachno
  • Multiple SoundFonts can be installed; the first one is used by default

Adding Files

Files can be added to the playlist in several ways:

  • File menu — Use File > Open (Cmd/Ctrl + O) to open the file picker dialog
  • Drag and drop — Drag media files from Finder or Explorer onto the player window
  • Eject button — Click the eject button in the transport controls to open the file picker

Auto-Play Behavior

When adding files, ONIXPlayer uses smart auto-play:

  • Single file — Always plays immediately
  • Multiple files + empty playlist — Plays from the first file
  • Multiple files + existing playlist — Appends without interrupting current playback

Playlist Sidebar

Press Tab to toggle the playlist sidebar on the right side of the window. The sidebar shows all tracks in the playlist and highlights the current track. Click any track to select and play it.

Shuffle and Repeat

Toggle shuffle and repeat modes using the buttons in the transport controls:

  • Shuffle — Randomizes playback order (Fisher-Yates shuffle)
  • Repeat — Loops the playlist after the last track finishes

Saving and Loading Playlists

ONIXPlayer can save and load playlists using the .opp (ONIXPlayer Playlist) format:

  • Save Playlist (Cmd/Ctrl + S) — Saves to the current file, or shows Save As dialog if no file is open
  • Save Playlist As (Cmd/Ctrl + Shift + S) — Always shows the save dialog
  • Open Playlist (Cmd/Ctrl + Shift + O) — Opens a .opp file, replacing the current playlist

Accessing Settings

Open Settings with Cmd + , on macOS or Ctrl + , on Windows/Linux, or via the View > Options menu item. The settings view replaces the player and the close button (macOS red traffic light) returns to the player.

Settings Categories

  • Application — Server port configuration, controls auto-hide delay
  • Appearance — Glass effect (window transparency), background color, and window tint controls with HSL/HSLA sliders
  • Dependencies — Install/uninstall FFmpeg, FluidSynth, and SoundFont files
  • Playback — Default volume, crossfade duration, previous track threshold, skip duration, and video aspect ratio
  • Transcoding — Video quality preset and audio bitrate for transcoded formats
  • Visualizations — Default visualization type, max frame rate, FFT size, and per-visualization customization

Glass Effect

On macOS and Windows, ONIXPlayer supports native window transparency with blur effects (vibrancy on macOS, acrylic on Windows). Enable this in Settings > Appearance. The glass effect requires an application restart to take effect.
